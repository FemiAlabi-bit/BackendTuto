Your best learning style for this project
Because you want practical experiment learning, I suggest this exact structure:

For each concept, always do 4 things
1. Read a short explanation
not 3 hours of theory
2. Find it in your codebase
real example from the project
3. Rewrite it in plain English
example:

“This middleware reads a token and stores email/admin info for later routes.”

4. Test it mentally or locally
If local run works, great.
If not, simulate the request flow on paper.

#################################################
What you should NOT do
Avoid these common traps:

❌ Trap 1 — learning too much theory before touching code
If you spend weeks only on generic backend theory, you’ll forget why you’re learning it.

❌ Trap 2 — trying to understand the entire codebase at once
That creates overwhelm.

❌ Trap 3 — trying to “master Go” before understanding the backend
You don’t need that yet.

❌ Trap 4 — comparing yourself to the external developer
That person likely built this over time with full context.
You are onboarding from the outside.
Different situation.
